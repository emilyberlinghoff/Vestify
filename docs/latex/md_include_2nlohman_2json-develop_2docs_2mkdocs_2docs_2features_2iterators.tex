\doxylink{class_a}{A} {\ttfamily \doxylink{classbasic__json}{basic\+\_\+json}} value is a container and allows access via iterators. Depending on the value type, {\ttfamily \doxylink{classbasic__json}{basic\+\_\+json}} stores zero or more values.

As for other containers, {\ttfamily begin()} returns an iterator to the first value and {\ttfamily end()} returns an iterator to the value following the last value. \doxylink{requirements_The}{The} latter iterator is a placeholder and cannot be dereferenced. In case of null values, empty arrays, or empty objects, {\ttfamily begin()} will return {\ttfamily end()}.

\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2iterators_autotoc_md1447}{}\doxysubsection{\texorpdfstring{Iteration order for objects}{Iteration order for objects}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2iterators_autotoc_md1447}
When iterating over objects, values are ordered with respect to the {\ttfamily object\+\_\+comparator\+\_\+t} type which defaults to {\ttfamily std::\+less}. See the \href{types/index.md\#key-order}{\texttt{types documentation}} for more information.

??? example \begin{DoxyVerb}```cpp
// create JSON object {"one": 1, "two": 2, "three": 3}
json j;
j["one"] = 1;
j["two"] = 2;
j["three"] = 3;

for (auto it = j.begin(); it != j.end(); ++it)
{
    std::cout << *it << std::endl;
}
```

Output:

```json
1
3
2
```

The reason for the order is the lexicographic ordering of the object keys "one", "three", "two".
\end{DoxyVerb}
\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2iterators_autotoc_md1448}{}\doxysubsection{\texorpdfstring{Access object keys during iteration}{Access object keys during iteration}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2iterators_autotoc_md1448}
\doxylink{requirements_The}{The} JSON iterators have two member functions, {\ttfamily key()} and {\ttfamily value()} to access the object key and stored value, respectively. When calling {\ttfamily key()} on a non-\/object iterator, an \href{../home/exceptions.md\#jsonexceptioninvalid_iterator207}{\texttt{invalid\+\_\+iterator.\+207}} exception is thrown.

??? example \begin{DoxyVerb}```cpp
// create JSON object {"one": 1, "two": 2, "three": 3}
json j;
j["one"] = 1;
j["two"] = 2;
j["three"] = 3;

for (auto it = j.begin(); it != j.end(); ++it)
{
    std::cout << it.key() << " : " << it.value() << std::endl;
}
```

Output:

```json
one : 1
three : 3
two : 2
```
\end{DoxyVerb}

\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2iterators_autotoc_md1450}{}\doxysubsection{\texorpdfstring{Reverse iteration order}{Reverse iteration order}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2iterators_autotoc_md1450}
{\ttfamily rbegin()} and {\ttfamily rend()} return iterators in the reverse sequence.



??? example \begin{DoxyVerb}```cpp
json j = {1, 2, 3, 4};

for (auto it = j.rbegin(); it != j.rend(); ++it)
{
    std::cout << *it << std::endl;
}
```

Output:

```json
4
3
2
1
```
\end{DoxyVerb}
\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2iterators_autotoc_md1451}{}\doxysubsection{\texorpdfstring{Iterating strings and binary values}{Iterating strings and binary values}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2iterators_autotoc_md1451}
Note that "{}value"{} means a JSON value in this setting, not values stored in the underlying containers. That is, {\ttfamily \texorpdfstring{$\ast$}{*}begin()} returns the complete string or binary array and is also safe the underlying string or binary array is empty.

??? example \begin{DoxyVerb}```cpp
json j = "Hello, world";
for (auto it = j.begin(); it != j.end(); ++it)
{
    std::cout << *it << std::endl;
}
```

Output:

```json
"Hello, world"
```
\end{DoxyVerb}
