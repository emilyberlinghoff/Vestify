\begin{DoxyEnumerate}
\item 3-\/way compares two JSON values producing a result of type {\ttfamily std::\+partial\+\_\+ordering} according to the following rules:\+
\begin{DoxyItemize}
\item Two JSON values compare with a result of {\ttfamily std::\+partial\+\_\+ordering::\+unordered} if either value is discarded.
\item If both JSON values are of the same type, the result is produced by 3-\/way comparing their stored values using their respective {\ttfamily operator\texorpdfstring{$<$}{<}=\texorpdfstring{$>$}{>}}.
\item Integer and floating-\/point numbers are converted to their common type and then 3-\/way compared using their respective {\ttfamily operator\texorpdfstring{$<$}{<}=\texorpdfstring{$>$}{>}}. For instance, comparing an integer and a floating-\/point value will 3-\/way compare the first value converted to floating-\/point with the second value.
\item Otherwise, yields a result by comparing the type (see \doxysectlink{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2basic__json_2value__t}{{\ttfamily value\+\_\+t}}{0}).
\end{DoxyItemize}
\item 3-\/way compares a JSON value and a scalar or a scalar and a JSON value by converting the scalar to a JSON value and 3-\/way comparing both JSON values (see 1).

!!! note "{}\+Comparing \`{}\+Na\+N\`{}"{} \begin{DoxyVerb}- `NaN` values are unordered within the domain of numbers.
  The following comparisons all yield `std::partial_ordering::unordered`:
    1. Comparing a `NaN` with itself.
    2. Comparing a `NaN` with another `NaN`.
    3. Comparing a `NaN` and any other number.
\end{DoxyVerb}
