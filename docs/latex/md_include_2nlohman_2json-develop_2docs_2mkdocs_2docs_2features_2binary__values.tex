\doxylink{requirements_The}{The} library implements several \doxysectlink{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2binary__formats_2index}{binary formats}{0} that encode JSON in an efficient way. Most of these formats support binary values; that is, values that have semantics defined outside the library and only define a sequence of bytes to be stored.

JSON itself does not have a binary value. As such, binary values are an extension that this library implements to store values received by a binary format. Binary values are never created by the JSON parser and are only part of a serialized JSON text if they have been created manually or via a binary format.\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2binary__values_autotoc_md1419}{}\doxysection{\texorpdfstring{API for binary values}{API for binary values}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2binary__values_autotoc_md1419}

\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2binary__values_autotoc_md1420}{}\doxysection{\texorpdfstring{Serialization}{Serialization}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2binary__values_autotoc_md1420}
Binary values are serialized differently according to the formats.\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2binary__values_autotoc_md1421}{}\doxysubsection{\texorpdfstring{JSON}{JSON}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2binary__values_autotoc_md1421}
JSON does not have a binary type, and this library does not introduce a new type as this would break conformance. Instead, binary values are serialized as an object with two keys:\+ {\ttfamily bytes} holds an array of integers, and {\ttfamily subtype} is an integer or {\ttfamily null}.

??? example \begin{DoxyVerb}Code:

```cpp
// create a binary value of subtype 42
json j;
j["binary"] = json::binary({0xCA, 0xFE, 0xBA, 0xBE}, 42);

// serialize to standard output
std::cout << j.dump(2) << std::endl;
```

Output:

```json
{
  "binary": {
    "bytes": [202, 254, 186, 190],
    "subtype": 42
  }
}
```
\end{DoxyVerb}


!!! warning "{}\+No roundtrip for binary values"{} \begin{DoxyVerb}The JSON parser will not parse the objects generated by binary values back to binary values. This is by design to
remain standards compliant. Serializing binary values to JSON is only implemented for debugging purposes.
\end{DoxyVerb}
\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2binary__values_autotoc_md1422}{}\doxysubsection{\texorpdfstring{BJData}{BJData}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2binary__values_autotoc_md1422}
\doxysectlink{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2binary__formats_2bjdata}{BJData}{0} neither supports binary values nor subtypes and proposes to serialize binary values as an array of uint8 values. \doxylink{requirements_The}{The} library implements this translation.

??? example \begin{DoxyVerb}Code:

```cpp
// create a binary value of subtype 42 (will be ignored in BJData)
json j;
j["binary"] = json::binary({0xCA, 0xFE, 0xBA, 0xBE}, 42);

// convert to BJData
auto v = json::to_bjdata(j);      
```

`v` is a `std::vector<std::uint8t>` with the following 20 elements:

```c
0x7B                                             // '{'
    0x69 0x06                                    // i 6 (length of the key)
    0x62 0x69 0x6E 0x61 0x72 0x79                // "binary"
    0x5B                                         // '['
        0x55 0xCA 0x55 0xFE 0x55 0xBA 0x55 0xBE  // content (each byte prefixed with 'U')
    0x5D                                         // ']'
0x7D                                             // '}'
```

The following code uses the type and size optimization for UBJSON:

```cpp
// convert to UBJSON using the size and type optimization
auto v = json::to_bjdata(j, true, true);
```

The resulting vector has 22 elements; the optimization is not effective for examples with few values:

```c
0x7B                                // '{'
    0x23 0x69 0x01                  // '#' 'i' type of the array elements: unsigned integers
    0x69 0x06                       // i 6 (length of the key)
    0x62 0x69 0x6E 0x61 0x72 0x79   // "binary"
    0x5B                            // '[' array
        0x24 0x55                   // '$' 'U' type of the array elements: unsigned integers
        0x23 0x69 0x04              // '#' i 4 number of array elements
        0xCA 0xFE 0xBA 0xBE         // content
```

Note that subtype (42) is **not** serialized and that UBJSON has **no binary type**, and deserializing `v` would
yield the following value:

```json
{
  "binary": [202, 254, 186, 190]
}
```
\end{DoxyVerb}
\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2binary__values_autotoc_md1423}{}\doxysubsection{\texorpdfstring{BSON}{BSON}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2binary__values_autotoc_md1423}
\doxysectlink{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2binary__formats_2bson}{BSON}{0} supports binary values and subtypes. If a subtype is given, it is used and added as an unsigned 8-\/bit integer. If no subtype is given, the generic binary subtype 0x00 is used.

??? example \begin{DoxyVerb}Code:

```cpp
// create a binary value of subtype 42
json j;
j["binary"] = json::binary({0xCA, 0xFE, 0xBA, 0xBE}, 42);

// convert to BSON
auto v = json::to_bson(j);      
```

`v` is a `std::vector<std::uint8t>` with the following 22 elements:

```c
0x16 0x00 0x00 0x00                         // number of bytes in the document
    0x05                                    // binary value
        0x62 0x69 0x6E 0x61 0x72 0x79 0x00  // key "binary" + null byte
        0x04 0x00 0x00 0x00                 // number of bytes
        0x2a                                // subtype
        0xCA 0xFE 0xBA 0xBE                 // content
0x00                                        // end of the document
```

Note that the serialization preserves the subtype, and deserializing `v` would yield the following value:

```json
{
  "binary": {
    "bytes": [202, 254, 186, 190],
    "subtype": 42
  }
}
```
\end{DoxyVerb}
\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2binary__values_autotoc_md1424}{}\doxysubsection{\texorpdfstring{CBOR}{CBOR}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2binary__values_autotoc_md1424}
\doxysectlink{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2binary__formats_2cbor}{CBOR}{0} supports binary values, but no subtypes. Subtypes will be serialized as tags. Any binary value will be serialized as byte strings. \doxylink{requirements_The}{The} library will choose the smallest representation using the length of the byte array.

??? example \begin{DoxyVerb}Code:

```cpp
// create a binary value of subtype 42
json j;
j["binary"] = json::binary({0xCA, 0xFE, 0xBA, 0xBE}, 42);

// convert to CBOR
auto v = json::to_cbor(j);      
```

`v` is a `std::vector<std::uint8t>` with the following 15 elements:

```c
0xA1                                   // map(1)
    0x66                               // text(6)
        0x62 0x69 0x6E 0x61 0x72 0x79  // "binary"
    0xD8 0x2A                          // tag(42)
    0x44                               // bytes(4)
        0xCA 0xFE 0xBA 0xBE            // content
```

Note that the subtype is serialized as tag. However, parsing tagged values yield a parse error unless
`json::cbor_tag_handler_t::ignore` or `json::cbor_tag_handler_t::store` is passed to `json::from_cbor`.

```json
{
  "binary": {
    "bytes": [202, 254, 186, 190],
    "subtype": null
  }
}
```
\end{DoxyVerb}
\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2binary__values_autotoc_md1425}{}\doxysubsection{\texorpdfstring{Message\+Pack}{Message\+Pack}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2binary__values_autotoc_md1425}
\doxysectlink{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2binary__formats_2messagepack}{Message\+Pack}{0} supports binary values and subtypes. If a subtype is given, the ext family is used. \doxylink{requirements_The}{The} library will choose the smallest representation among fixext1, fixext2, fixext4, fixext8, ext8, ext16, and ext32. \doxylink{requirements_The}{The} subtype is then added as a signed 8-\/bit integer.

If no subtype is given, the bin family (bin8, bin16, bin32) is used.

??? example \begin{DoxyVerb}Code:

```cpp
// create a binary value of subtype 42
json j;
j["binary"] = json::binary({0xCA, 0xFE, 0xBA, 0xBE}, 42);

// convert to MessagePack
auto v = json::to_msgpack(j);      
```

`v` is a `std::vector<std::uint8t>` with the following 14 elements:

```c
0x81                                   // fixmap1
    0xA6                               // fixstr6
        0x62 0x69 0x6E 0x61 0x72 0x79  // "binary"
    0xD6                               // fixext4
        0x2A                           // subtype
        0xCA 0xFE 0xBA 0xBE            // content
```

Note that the serialization preserves the subtype, and deserializing `v` would yield the following value:

```json
{
  "binary": {
    "bytes": [202, 254, 186, 190],
    "subtype": 42
  }
}
```
\end{DoxyVerb}
