These macros can be used to simplify the serialization/\+deserialization of derived types if you want to use a JSON object as serialization and want to use the member variable names as object keys in that object.


\begin{DoxyItemize}
\item Macros 1, 2, and 3 are to be defined {\bfseries{inside}} the class/\+struct to create code for. Like \doxysectlink{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2macros_2nlohmann__define__type__intrusive}{{\ttfamily NLOHMANN\+\_\+\+DEFINE\+\_\+\+TYPE\+\_\+\+INTRUSIVE}}{0}, they can access private members.
\item Macros 4, 5, and 6 are to be defined {\bfseries{outside}} the class/\+struct to create code for, but {\bfseries{inside}} its namespace. Like \doxysectlink{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2macros_2nlohmann__define__type__non__intrusive}{{\ttfamily NLOHMANN\+\_\+\+DEFINE\+\_\+\+TYPE\+\_\+\+NON\+\_\+\+INTRUSIVE}}{0}, they {\bfseries{cannot}} access private members.
\end{DoxyItemize}

\doxylink{requirements_The}{The} first parameter is the name of the derived class/\+struct, the second parameter is the name of the base class/\+struct and all remaining parameters name the members. \doxylink{requirements_The}{The} base type {\bfseries{must}} be already serializable/\+deserializable.


\begin{DoxyItemize}
\item Macros 1 and 4 will use \doxysectlink{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2basic__json_2at}{{\ttfamily at}}{0} during deserialization and will throw \href{../../home/exceptions.md\#jsonexceptionout_of_range403}{\texttt{{\ttfamily out\+\_\+of\+\_\+range.\+403}}} if a key is missing in the JSON object.
\item Macros 2 and 5 will use \doxysectlink{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2basic__json_2value}{{\ttfamily value}}{0} during deserialization and fall back to the default value for the respective type of the member variable if a key in the JSON object is missing. \doxylink{requirements_The}{The} generated {\ttfamily from\+\_\+json()} function default constructs an object and uses its values as the defaults when calling the {\ttfamily value} function.
\end{DoxyItemize}

- `A` has private members and is not a derived class. Hence, macro `NLOHMANN_DEFINE_TYPE_INTRUSIVE` is used.

  `NLOHMANN_DEFINE_DERIVED_TYPE_INTRUSIVE` must be used.

  `NLOHMANN_DEFINE_TYPE_INTRUSIVE`.
