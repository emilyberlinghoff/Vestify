\doxylink{requirements_The}{The} type used to store JSON strings.

\href{https://tools.ietf.org/html/rfc8259}{\texttt{RFC 8259}} describes JSON strings as follows:\+ \begin{quote}
\doxylink{class_a}{A} string is a sequence of zero or more Unicode characters. \end{quote}


To store objects in C++, a type is defined by the template parameter described below. Unicode values are split by the JSON class into byte-\/sized characters during deserialization.\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2basic__json_2string__t_autotoc_md862}{}\doxysection{\texorpdfstring{Template parameters}{Template parameters}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2basic__json_2string__t_autotoc_md862}

Note this container is used for keys/\+names in objects, see \doxysectlink{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2basic__json_2object__t}{object\+\_\+t}{0}.\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2basic__json_2string__t_autotoc_md863}{}\doxysection{\texorpdfstring{Notes}{Notes}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2basic__json_2string__t_autotoc_md863}

With the default values for {\ttfamily String\+Type} ({\ttfamily std::\+string}), the default value for {\ttfamily string\+\_\+t} is {\ttfamily \#!cpp std::\+string}.\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2basic__json_2string__t_autotoc_md865}{}\doxysubsubsection{\texorpdfstring{Encoding}{Encoding}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2basic__json_2string__t_autotoc_md865}
Strings are stored in UTF-\/8 encoding. Therefore, functions like {\ttfamily std::\+string::\+size()} or {\ttfamily std::\+string::\+length()} return the number of bytes in the string rather than the number of characters or glyphs.\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2basic__json_2string__t_autotoc_md866}{}\doxysubsubsection{\texorpdfstring{String comparison}{String comparison}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2basic__json_2string__t_autotoc_md866}
\href{https://tools.ietf.org/html/rfc8259}{\texttt{RFC 8259}} states:\+ \begin{quote}
Software implementations are typically required to test names of object members for equality. Implementations that transform the textual representation into sequences of Unicode code units and then perform the comparison numerically, code unit by code unit, are interoperable in the sense that implementations will agree in all cases on equality or inequality of two strings. For example, implementations that compare strings with escaped characters unconverted may incorrectly find that {\ttfamily "{}a\textbackslash{}\textbackslash{}b"{}} and {\ttfamily "{}a\textbackslash{}u005\+Cb"{}} are not equal. \end{quote}


This implementation is interoperable as it does compare strings code unit by code unit.\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2basic__json_2string__t_autotoc_md867}{}\doxysubsubsection{\texorpdfstring{Storage}{Storage}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2basic__json_2string__t_autotoc_md867}
String values are stored as pointers in a {\ttfamily \doxylink{classbasic__json}{basic\+\_\+json}} type. That is, for any access to string values, a pointer of type {\ttfamily string\+\_\+t\texorpdfstring{$\ast$}{*}} must be dereferenced.\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2basic__json_2string__t_autotoc_md868}{}\doxysection{\texorpdfstring{Examples}{Examples}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2basic__json_2string__t_autotoc_md868}
