When defined, headers {\ttfamily \texorpdfstring{$<$}{<}cstdio\texorpdfstring{$>$}{>}}, {\ttfamily \texorpdfstring{$<$}{<}ios\texorpdfstring{$>$}{>}}, {\ttfamily \texorpdfstring{$<$}{<}iosfwd\texorpdfstring{$>$}{>}}, {\ttfamily \texorpdfstring{$<$}{<}istream\texorpdfstring{$>$}{>}}, and {\ttfamily \texorpdfstring{$<$}{<}ostream\texorpdfstring{$>$}{>}} are not included and parse functions relying on these headers are excluded. This is relevant for environments where these I/\+O functions are disallowed for security reasons (e.\+g., Intel Software Guard Extensions (SGX)).\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2macros_2json__no__io_autotoc_md1227}{}\doxysection{\texorpdfstring{Default definition}{Default definition}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2macros_2json__no__io_autotoc_md1227}
By default, {\ttfamily \#!cpp JSON\+\_\+\+NO\+\_\+\+IO} is not defined.
