\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2home_2exceptions_autotoc_md1580}{}\doxysubsection{\texorpdfstring{Switch off exceptions}{Switch off exceptions}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2home_2exceptions_autotoc_md1580}
Exceptions are used widely within the library. They can, however, be switched off with either using the compiler flag {\ttfamily -\/fno-\/exceptions} or by defining the symbol \doxysectlink{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2macros_2json__noexception}{{\ttfamily JSON\+\_\+\+NOEXCEPTION}}{0}. In this case, exceptions are replaced by {\ttfamily abort()} calls. You can further control this behavior by defining {\ttfamily JSON\+\_\+\+THROW\+\_\+\+USER} (overriding {\ttfamily \#!cpp throw}), {\ttfamily JSON\+\_\+\+TRY\+\_\+\+USER} (overriding {\ttfamily \#!cpp try}), and {\ttfamily JSON\+\_\+\+CATCH\+\_\+\+USER} (overriding {\ttfamily \#!cpp catch}).

Note that \doxysectlink{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2macros_2json__throw__user}{{\ttfamily JSON\+\_\+\+THROW\+\_\+\+USER}}{0} should leave the current scope (e.\+g., by throwing or aborting), as continuing after it may yield undefined behavior.

??? example \begin{DoxyVerb}The code below switches off exceptions and creates a log entry with a detailed error message in case of errors.

```cpp
#include <iostream>

#define JSON_TRY_USER if(true)
#define JSON_CATCH_USER(exception) if(false)
#define JSON_THROW_USER(exception)                           \
    {std::clog << "Error in " << __FILE__ << ":" << __LINE__ \
               << " (function " << __FUNCTION__ << ") - "    \
               << (exception).what() << std::endl;           \
     std::abort();}

#include <nlohmann/json.hpp>
```
\end{DoxyVerb}


Note the explanatory \href{https://en.cppreference.com/w/cpp/error/exception/what}{\texttt{{\ttfamily what()}}} string of exceptions is not available for MSVC if exceptions are disabled, see \href{https://github.com/nlohmann/json/discussions/2824}{\texttt{\#2824}}.

See \doxysectlink{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2macros_2json__throw__user}{documentation of {\ttfamily JSON\+\_\+\+TRY\+\_\+\+USER}, {\ttfamily JSON\+\_\+\+CATCH\+\_\+\+USER} and {\ttfamily JSON\+\_\+\+THROW\+\_\+\+USER}}{0} for more information.\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2home_2exceptions_autotoc_md1581}{}\doxysubsection{\texorpdfstring{Extended diagnostic messages}{Extended diagnostic messages}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2home_2exceptions_autotoc_md1581}

To create better diagnostics messages, each JSON value needs a pointer to its parent value such that a global context (i.\+e., a path from the root value to the value that led to the exception) can be created. That global context is provided as \doxysectlink{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2json__pointer}{JSON Pointer}{0}.

As this global context comes at the price of storing one additional pointer per JSON value and runtime overhead to maintain the parent relation, extended diagnostics are disabled by default. They can, however, be enabled by defining the preprocessor symbol \doxysectlink{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2macros_2json__diagnostics}{{\ttfamily JSON\+\_\+\+DIAGNOSTICS}}{0} to {\ttfamily 1} before including {\ttfamily json.\+hpp}.

```

\end{DoxyVerb}

Exceptions have ids 1xx.

!!! info "{}\+Byte index"{} \begin{DoxyVerb}Member `byte` holds the byte index of the last read character in the input
file.

For an input with n bytes, 1 is the index of the first character and n+1
is the index of the terminating null byte or the end of file. This also
holds true when reading a byte vector (CBOR or MessagePack).
\end{DoxyVerb}
