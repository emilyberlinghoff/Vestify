This macro indicates whether the standard library has any support for RTTI (run time type information). Possible values are {\ttfamily 1} when supported or {\ttfamily 0} when unsupported.\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2macros_2json__has__static__rtti_autotoc_md1219}{}\doxysection{\texorpdfstring{Default definition}{Default definition}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2macros_2json__has__static__rtti_autotoc_md1219}
\doxylink{requirements_The}{The} default value is detected based on the preprocessor macro {\ttfamily \#!cpp \+\_\+\+HAS\+\_\+\+STATIC\+\_\+\+RTTI}.

When the macro is not defined, the library will define it to its default value.\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2macros_2json__has__static__rtti_autotoc_md1220}{}\doxysection{\texorpdfstring{Examples}{Examples}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2macros_2json__has__static__rtti_autotoc_md1220}
??? example \begin{DoxyVerb}The code below forces the library to enable support for libraries with RTTI dependence:

```cpp
#define JSON_HAS_STATIC_RTTI 1
#include <nlohmann/json.hpp>

...
```
\end{DoxyVerb}
