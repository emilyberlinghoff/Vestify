This macro enables extended diagnostics for exception messages. Possible values are {\ttfamily 1} to enable or {\ttfamily 0} to disable (default).

When enabled, exception messages contain a \doxysectlink{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2json__pointer}{JSON Pointer}{0} to the JSON value that triggered the exception, see \href{../home/exceptions.md\#extended-diagnostic-messages}{\texttt{Extended diagnostic messages}} for an example. Note that enabling this macro increases the size of every JSON value by one pointer and adds some runtime overhead.

\doxylink{requirements_The}{The} diagnostics messages can also be controlled with the CMake option \href{../integration/cmake.md\#json_diagnostics}{\texttt{{\ttfamily JSON\+\_\+\+Diagnostics}}} ({\ttfamily OFF} by default) which sets {\ttfamily JSON\+\_\+\+DIAGNOSTICS} accordingly.

See \doxysectlink{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2macros_2json__diagnostics}{full documentation of {\ttfamily JSON\+\_\+\+DIAGNOSTICS}}{0}.\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2macros_autotoc_md1466}{}\doxysection{\texorpdfstring{{\ttfamily JSON\+\_\+\+DIAGNOSTIC\+\_\+\+POSITIONS}}{{\ttfamily JSON\+\_\+\+DIAGNOSTIC\+\_\+\+POSITIONS}}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2macros_autotoc_md1466}
When enabled, two new member functions \doxysectlink{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2basic__json_2start__pos}{{\ttfamily start\+\_\+pos()}}{0} and \doxysectlink{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2basic__json_2end__pos}{{\ttfamily end\+\_\+pos()}}{0} are added to \doxysectlink{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2basic__json_2index}{{\ttfamily basic\+\_\+json}}{0} values. If the value was created by calling the\doxysectlink{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2basic__json_2parse}{{\ttfamily parse}}{0} function, then these functions allow querying the byte positions of the value in the input it was parsed from. \doxylink{requirements_The}{The} byte positions are also used in exceptions to help locate errors.

\doxylink{requirements_The}{The} diagnostics positions can also be controlled with the CMake option \href{../integration/cmake.md\#json_diagnostic_positions}{\texttt{{\ttfamily JSON\+\_\+\+Diagnostic\+\_\+\+Positions}}} ({\ttfamily OFF} by default) which sets {\ttfamily JSON\+\_\+\+DIAGNOSTIC\+\_\+\+POSITIONS} accordingly.

See \doxysectlink{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2macros_2json__diagnostic__positions}{full documentation of {\ttfamily JSON\+\_\+\+DIAGNOSTIC\+\_\+\+POSITIONS}}{0}\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2macros_autotoc_md1467}{}\doxysection{\texorpdfstring{{\ttfamily JSON\+\_\+\+HAS\+\_\+\+CPP\+\_\+11}, {\ttfamily JSON\+\_\+\+HAS\+\_\+\+CPP\+\_\+14}, {\ttfamily JSON\+\_\+\+HAS\+\_\+\+CPP\+\_\+17}, {\ttfamily JSON\+\_\+\+HAS\+\_\+\+CPP\+\_\+20}}{{\ttfamily JSON\+\_\+\+HAS\+\_\+\+CPP\+\_\+11}, {\ttfamily JSON\+\_\+\+HAS\+\_\+\+CPP\+\_\+14}, {\ttfamily JSON\+\_\+\+HAS\+\_\+\+CPP\+\_\+17}, {\ttfamily JSON\+\_\+\+HAS\+\_\+\+CPP\+\_\+20}}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2macros_autotoc_md1467}
\doxylink{requirements_The}{The} library targets C++11, but also supports some features introduced in later C++ versions (e.\+g., {\ttfamily std::\+string\+\_\+view} support for C++17). For these new features, the library implements some preprocessor checks to determine the C++ standard. By defining any of these symbols, the internal check is overridden and the provided C++ version is unconditionally assumed. This can be helpful for compilers that only implement parts of the standard and would be detected incorrectly.

See \doxysectlink{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2macros_2json__has__cpp__11}{full documentation of {\ttfamily JSON\+\_\+\+HAS\+\_\+\+CPP\+\_\+11}, {\ttfamily JSON\+\_\+\+HAS\+\_\+\+CPP\+\_\+14}, {\ttfamily JSON\+\_\+\+HAS\+\_\+\+CPP\+\_\+17}, and {\ttfamily JSON\+\_\+\+HAS\+\_\+\+CPP\+\_\+20}}{0}.\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2macros_autotoc_md1468}{}\doxysection{\texorpdfstring{{\ttfamily JSON\+\_\+\+HAS\+\_\+\+FILESYSTEM}, {\ttfamily JSON\+\_\+\+HAS\+\_\+\+EXPERIMENTAL\+\_\+\+FILESYSTEM}}{{\ttfamily JSON\+\_\+\+HAS\+\_\+\+FILESYSTEM}, {\ttfamily JSON\+\_\+\+HAS\+\_\+\+EXPERIMENTAL\+\_\+\+FILESYSTEM}}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2macros_autotoc_md1468}
When compiling with C++17, the library provides conversions from and to {\ttfamily std::\+filesystem::\+path}. As compiler support for filesystem is limited, the library tries to detect whether {\ttfamily \texorpdfstring{$<$}{<}filesystem\texorpdfstring{$>$}{>}}/\+{\ttfamily std::\+filesystem} ({\ttfamily JSON\+\_\+\+HAS\+\_\+\+FILESYSTEM}) or {\ttfamily \texorpdfstring{$<$}{<}experimental/\+filesystem\texorpdfstring{$>$}{>}}/\+{\ttfamily std::\+experimental::\+filesystem} ({\ttfamily JSON\+\_\+\+HAS\+\_\+\+EXPERIMENTAL\+\_\+\+FILESYSTEM}) should be used. To override the built-\/in check, define {\ttfamily JSON\+\_\+\+HAS\+\_\+\+FILESYSTEM} or {\ttfamily JSON\+\_\+\+HAS\+\_\+\+EXPERIMENTAL\+\_\+\+FILESYSTEM} to {\ttfamily 1}.

See \doxysectlink{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2macros_2json__has__filesystem}{full documentation of {\ttfamily JSON\+\_\+\+HAS\+\_\+\+FILESYSTEM} and {\ttfamily JSON\+\_\+\+HAS\+\_\+\+EXPERIMENTAL\+\_\+\+FILESYSTEM}}{0}.\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2macros_autotoc_md1469}{}\doxysection{\texorpdfstring{{\ttfamily JSON\+\_\+\+NOEXCEPTION}}{{\ttfamily JSON\+\_\+\+NOEXCEPTION}}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2macros_autotoc_md1469}
Exceptions can be switched off by defining the symbol {\ttfamily JSON\+\_\+\+NOEXCEPTION}.

See \doxysectlink{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2macros_2json__noexception}{full documentation of {\ttfamily JSON\+\_\+\+NOEXCEPTION}}{0}.\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2macros_autotoc_md1470}{}\doxysection{\texorpdfstring{{\ttfamily JSON\+\_\+\+DISABLE\+\_\+\+ENUM\+\_\+\+SERIALIZATION}}{{\ttfamily JSON\+\_\+\+DISABLE\+\_\+\+ENUM\+\_\+\+SERIALIZATION}}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2macros_autotoc_md1470}

When defined, headers {\ttfamily \texorpdfstring{$<$}{<}cstdio\texorpdfstring{$>$}{>}}, {\ttfamily \texorpdfstring{$<$}{<}ios\texorpdfstring{$>$}{>}}, {\ttfamily \texorpdfstring{$<$}{<}iosfwd\texorpdfstring{$>$}{>}}, {\ttfamily \texorpdfstring{$<$}{<}istream\texorpdfstring{$>$}{>}}, and {\ttfamily \texorpdfstring{$<$}{<}ostream\texorpdfstring{$>$}{>}} are not included and parse functions relying on these headers are excluded. When defined, the library will not create a compiler warning when a different version of the library was already included.

See \doxysectlink{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2macros_2json__skip__library__version__check}{full documentation of {\ttfamily JSON\+\_\+\+SKIP\+\_\+\+LIBRARY\+\_\+\+VERSION\+\_\+\+CHECK}}{0}.\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2macros_autotoc_md1473}{}\doxysection{\texorpdfstring{{\ttfamily JSON\+\_\+\+SKIP\+\_\+\+UNSUPPORTED\+\_\+\+COMPILER\+\_\+\+CHECK}}{{\ttfamily JSON\+\_\+\+SKIP\+\_\+\+UNSUPPORTED\+\_\+\+COMPILER\+\_\+\+CHECK}}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2macros_autotoc_md1473}
