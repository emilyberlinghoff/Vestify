This macro enables the (incorrect) legacy comparison behavior of discarded JSON values. Possible values are {\ttfamily 1} to enable or {\ttfamily 0} to disable (default).

When enabled, comparisons involving at least one discarded JSON value yield results as follows:\+


\begin{DoxyTable}{2}{}{}{1}
\SetCell{c,bg=\tableheadbgcolor,font=\bfseries}{\bfseries{Operator}}  &\SetCell{c,bg=\tableheadbgcolor,font=\bfseries}{\bfseries{Result}}  \\
{\ttfamily ==}  &{\ttfamily \#!cpp false}  \\
{\ttfamily !=}  &{\ttfamily \#!cpp true}  \\
{\ttfamily \texorpdfstring{$<$}{<}}  &{\ttfamily \#!cpp false}  \\
{\ttfamily \texorpdfstring{$<$}{<}=}  &{\ttfamily \#!cpp true}  \\
{\ttfamily \texorpdfstring{$>$}{>}=}  &{\ttfamily \#!cpp true}  \\
{\ttfamily \texorpdfstring{$>$}{>}}  &{\ttfamily \#!cpp false}  \\
\end{DoxyTable}


Otherwise, comparisons involving at least one discarded JSON value always yield {\ttfamily \#!cpp false}.\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2macros_2json__use__legacy__discarded__value__comparison_autotoc_md1264}{}\doxysection{\texorpdfstring{Default definition}{Default definition}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2macros_2json__use__legacy__discarded__value__comparison_autotoc_md1264}
\doxylink{requirements_The}{The} default value is {\ttfamily 0}.


\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{preprocessor}{\#define\ JSON\_USE\_LEGACY\_DISCARDED\_VALUE\_COMPARISON\ 0}}

\end{DoxyCode}


When the macro is not defined, the library will define it to its default value.\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2macros_2json__use__legacy__discarded__value__comparison_autotoc_md1265}{}\doxysection{\texorpdfstring{Notes}{Notes}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2macros_2json__use__legacy__discarded__value__comparison_autotoc_md1265}
!!! warning "{}\+Inconsistent behavior in C++20 and beyond"{} \begin{DoxyVerb}When targeting C++20 or above, enabling the legacy comparison behavior is _strongly_
discouraged.

  - The 3-way comparison operator (`<=>`) will always give the correct result
    (`#!cpp std::partial_ordering::unordered`) regardless of the value of
    `JSON_USE_LEGACY_DISCARDED_VALUE_COMPARISON`.
  - Overloads for the equality and relational operators emulate the legacy behavior.

Code outside your control may use either 3-way comparison or the equality and relational operators, resulting in
inconsistent and unpredictable behavior.

See [`operator<=>`](../basic_json/operator_spaceship.md) for more information on 3-way comparison.
\end{DoxyVerb}


!!! warning "{}\+Deprecation"{} \begin{DoxyVerb}The legacy comparison behavior is deprecated and may be removed in a future major version release.

New code should not depend on it and existing code should try to remove or rewrite expressions relying on it.
\end{DoxyVerb}


!!! hint "{}\+CMake option"{} \begin{DoxyVerb}Legacy comparison can also be controlled with the CMake option
[`JSON_LegacyDiscardedValueComparison`](../../integration/cmake.md#json_implicitconversions)
(`OFF` by default) which defines `JSON_USE_LEGACY_DISCARDED_VALUE_COMPARISON` accordingly.
\end{DoxyVerb}
