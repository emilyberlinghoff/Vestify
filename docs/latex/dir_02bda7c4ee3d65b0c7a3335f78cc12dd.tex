{\ttfamily amalgamate.\+py} aims to make it easy to use SQLite-\/style C source and header amalgamation in projects.

For more information, please refer to:\+ \href{http://sqlite.org/amalgamation.html}{\texttt{http:\+/\+/\+sqlite.\+org/\+amalgamation.\+html}}\hypertarget{dir_02bda7c4ee3d65b0c7a3335f78cc12dd_autotoc_md1947}{}\doxysubsection{\texorpdfstring{Here be dragons}{Here be dragons}}\label{dir_02bda7c4ee3d65b0c7a3335f78cc12dd_autotoc_md1947}
{\ttfamily amalgamate.\+py} is quite dumb, it only knows the bare minimum about C code required in order to be able to handle trivial include directives. It can produce weird results for unexpected code.

Things to be aware of:\+

{\ttfamily amalgamate.\+py} will not handle complex include directives correctly:\+ \begin{DoxyVerb}    #define HEADER_PATH "path/to/header.h"
    #include HEADER_PATH
\end{DoxyVerb}


In the above example, {\ttfamily path/\+to/\+header.\+h} will not be included in the amalgamation (HEADER\+\_\+\+PATH is never expanded).

{\ttfamily amalgamate.\+py} makes the assumption that each source and header file which is not empty will end in a new-\/line character, which is not immediately preceded by a backslash character (see 5.\+1.\+1.\+2p1.\+2 of ISO C99).

{\ttfamily amalgamate.\+py} should be usable with C++ code, but raw string literals from C++11 will definitely cause problems:\+ \begin{DoxyVerb}    R"delimiter(Terrible raw \ data " #include <sneaky.hpp>)delimiter"
    R"delimiter(Terrible raw \ data " escaping)delimiter"
\end{DoxyVerb}


In the examples above, {\ttfamily amalgamate.\+py} will stop parsing the raw string literal when it encounters the first quotation mark, which will produce unexpected results.\hypertarget{dir_02bda7c4ee3d65b0c7a3335f78cc12dd_autotoc_md1948}{}\doxysubsection{\texorpdfstring{Installing amalgamate.\+py}{Installing amalgamate.\+py}}\label{dir_02bda7c4ee3d65b0c7a3335f78cc12dd_autotoc_md1948}
Python v.\+2.\+7.\+0 or higher is required.

{\ttfamily amalgamate.\+py} can be tested and installed using the following commands:\+ \begin{DoxyVerb}    ./test.sh && sudo -k cp ./amalgamate.py /usr/local/bin/
\end{DoxyVerb}

\begin{DoxyItemize}
\item \doxylink{requirements_The}{The} {\ttfamily -\/c, -\/-\/config} option should specify the path to a JSON config file which lists the source files, include paths and where to write the resulting amalgamation. Have a look at {\ttfamily test/\+source.\+c.\+json} and {\ttfamily test/\+include.\+h.\+json} to see two examples.
\item \doxylink{requirements_The}{The} {\ttfamily -\/s, -\/-\/source} option should specify the path to the source directory. This is useful for supporting separate source and build directories.
\item \doxylink{requirements_The}{The} {\ttfamily -\/p, -\/-\/prologue} option should specify the path to a file which will be added to the beginning of the amalgamation. It is optional. 
\end{DoxyItemize}
