By default, enum values are serialized to JSON as integers. In some cases, this could result in undesired behavior. If an enum is modified or re-\/ordered after data has been serialized to JSON, the later deserialized JSON data may be undefined or a different enum value than was originally intended.

\doxylink{requirements_The}{The} {\ttfamily NLOHMANN\+\_\+\+JSON\+\_\+\+SERIALIZE\+\_\+\+ENUM} allows to define a user-\/defined serialization for every enumerator.\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2macros_2nlohmann__json__serialize__enum_autotoc_md1305}{}\doxysection{\texorpdfstring{Parameters}{Parameters}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2macros_2nlohmann__json__serialize__enum_autotoc_md1305}
{\ttfamily type} (in) :\+ name of the enum to serialize/\+deserialize

{\ttfamily conversion} (in) :\+ a pair of an enumerator and a JSON serialization; arbitrary pairs can be given as a comma-\/separated list\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2macros_2nlohmann__json__serialize__enum_autotoc_md1306}{}\doxysection{\texorpdfstring{Default definition}{Default definition}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2macros_2nlohmann__json__serialize__enum_autotoc_md1306}

!!! info "{}\+Prerequisites"{} \begin{DoxyVerb}The macro must be used inside the namespace of the enum.
\end{DoxyVerb}


!!! important "{}\+Important notes"{} \begin{DoxyVerb}- When using [`get<ENUM_TYPE>()`](../basic_json/get.md), undefined JSON values will default to the first specified
  conversion. Select this default pair carefully. See example 1 below.
- If an enum or JSON value is specified in multiple conversions, the first matching conversion from the top of the
  list will be returned when converting to or from JSON. See example 2 below.
\end{DoxyVerb}
