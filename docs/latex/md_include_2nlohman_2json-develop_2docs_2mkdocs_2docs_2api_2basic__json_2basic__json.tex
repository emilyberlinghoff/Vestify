\item Creates a JSON value of type array or object from the passed initializer list {\ttfamily init}. In case {\ttfamily type\+\_\+deduction} is {\ttfamily \#!cpp true} (default), the type of the JSON value to be created is deducted from the initializer list {\ttfamily init} according to the following rules:\+
\begin{DoxyEnumerate}
\item If the list is empty, an empty JSON object value {\ttfamily \{\}} is created.
\item If the list consists of pairs whose first element is a string, a JSON object value is created where the first elements of the pairs are treated as keys and the second elements are as values.
\item In all other cases, an array is created.
\end{DoxyEnumerate}

\doxylink{requirements_The}{The} rules aim to create the best fit between a C++ initializer list and JSON values. \doxylink{requirements_The}{The} rationale is as follows:\+
\begin{DoxyEnumerate}
\item \doxylink{requirements_The}{The} empty initializer list is written as {\ttfamily \#!cpp \{\}} which is exactly an empty JSON object.
\item C++ has no way of describing mapped types other than to list a list of pairs. As JSON requires that keys must be of type string, rule 2 is the weakest constraint one can pose on initializer lists to interpret them as an object.
\item In all other cases, the initializer list could not be interpreted as a JSON object type, so interpreting it as a JSON array type is safe.
\end{DoxyEnumerate}

With the rules described above, the following JSON values cannot be expressed by an initializer list:\+
\begin{DoxyItemize}
\item the empty array ({\ttfamily \#!json \+[]\+}):\+ use {\ttfamily \doxylink{classarray}{array(initializer\+\_\+list\+\_\+t)}} with an empty initializer list in this case
\item arrays whose elements satisfy rule 2:\+ use {\ttfamily \doxylink{classarray}{array(initializer\+\_\+list\+\_\+t)}} with the same initializer list in this case
\end{DoxyItemize}
