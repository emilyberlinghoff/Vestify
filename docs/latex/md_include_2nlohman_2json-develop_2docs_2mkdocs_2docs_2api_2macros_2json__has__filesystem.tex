When compiling with C++17, the library provides conversions from and to \href{https://en.cppreference.com/w/cpp/filesystem/path}{\texttt{{\ttfamily std::\+filesystem::\+path}}}. As compiler support for filesystem is limited, the library tries to detect whether \href{https://en.cppreference.com/w/cpp/header/filesystem}{\texttt{{\ttfamily \texorpdfstring{$<$}{<}filesystem\texorpdfstring{$>$}{>}}/\+{\ttfamily std::\+filesystem}}} ({\ttfamily JSON\+\_\+\+HAS\+\_\+\+FILESYSTEM}) or \href{https://en.cppreference.com/w/cpp/header/experimental/filesystem}{\texttt{{\ttfamily \texorpdfstring{$<$}{<}experimental/\+filesystem\texorpdfstring{$>$}{>}}/\+{\ttfamily std::\+experimental::\+filesystem}}} ({\ttfamily JSON\+\_\+\+HAS\+\_\+\+EXPERIMENTAL\+\_\+\+FILESYSTEM}) should be used. To override the built-\/in check, define {\ttfamily JSON\+\_\+\+HAS\+\_\+\+FILESYSTEM} or {\ttfamily JSON\+\_\+\+HAS\+\_\+\+EXPERIMENTAL\+\_\+\+FILESYSTEM} to {\ttfamily 1}.\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2macros_2json__has__filesystem_autotoc_md1210}{}\doxysection{\texorpdfstring{Default definition}{Default definition}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2macros_2json__has__filesystem_autotoc_md1210}
