\begin{DoxyItemize}
\item \doxylink{requirements_The}{The} library uses numerous assertions to guarantee invariants and to abort in case of otherwise undefined behavior (e.\+g., when calling operator\+[]\+ with a missing object key on a {\ttfamily const} object). See page \doxysectlink{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2assertions}{runtime assertions}{0} for more information.
\item Defining the macro to code that does not call {\ttfamily std::\+abort} may leave the library in an undefined state.
\item \doxylink{requirements_The}{The} macro is undefined outside the library.
\end{DoxyItemize}\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2macros_2json__assert_autotoc_md1183}{}\doxysection{\texorpdfstring{Examples}{Examples}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2macros_2json__assert_autotoc_md1183}
??? example "{}\+Example 1:\+ default behavior"{} \begin{DoxyVerb}The following code will trigger an assertion at runtime:

```cpp
#include <nlohmann/json.hpp>

using json = nlohmann::json;

int main()
{
    const json j = {{"key", "value"}};
    auto v = j["missing"];
}
```

Output:

```
Assertion failed: (m_value.object->find(key) != m_value.object->end()), function operator[], file json.hpp, line 2144.
```
\end{DoxyVerb}


??? example "{}\+Example 2:\+ user-\/defined behavior"{} \begin{DoxyVerb}The assertion reporting can be changed by defining `JSON_ASSERT(x)` differently.

```cpp
#include <cstdio>
#include <cstdlib>
#define JSON_ASSERT(x) if(!(x)){fprintf(stderr, "assertion error in %s\n", __FUNCTION__); std::abort();}

#include <nlohmann/json.hpp>

using json = nlohmann::json;

int main()
{
    const json j = {{"key", "value"}};
    auto v = j["missing"];
}
```

Output:

```
assertion error in operator[]
```
\end{DoxyVerb}
