\doxylink{requirements_The}{The} code contains numerous debug assertions to ensure class invariants are valid or to detect undefined behavior. Whereas the former class invariants are nothing to be concerned with, the latter checks for undefined behavior are to detect bugs in client code.\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2assertions_autotoc_md1389}{}\doxysection{\texorpdfstring{Switch off runtime assertions}{Switch off runtime assertions}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2assertions_autotoc_md1389}
Runtime assertions can be switched off by defining the preprocessor macro {\ttfamily NDEBUG} (see the \href{https://en.cppreference.com/w/cpp/error/assert}{\texttt{documentation of assert}}) which is the default for release builds.\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2assertions_autotoc_md1390}{}\doxysection{\texorpdfstring{Change assertion behavior}{Change assertion behavior}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2assertions_autotoc_md1390}
\doxylink{requirements_The}{The} behavior of runtime assertions can be changes by defining macro \doxysectlink{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2macros_2json__assert}{{\ttfamily JSON\+\_\+\+ASSERT(x)}}{0} before including the {\ttfamily json.\+hpp} header.\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2assertions_autotoc_md1391}{}\doxysection{\texorpdfstring{Function with runtime assertions}{Function with runtime assertions}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2assertions_autotoc_md1391}
\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2assertions_autotoc_md1392}{}\doxysubsection{\texorpdfstring{Unchecked object access to a const value}{Unchecked object access to a const value}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2assertions_autotoc_md1392}
Function {\ttfamily operator\+[]\+} implements unchecked access for objects. Whereas a missing key is added in the case of non-\/const objects, accessing a const object with a missing key is undefined behavior (think of a dereferenced null pointer) and yields a runtime assertion.

If you are not sure whether an element in an object exists, use checked access with the \doxysectlink{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2basic__json_2at}{{\ttfamily at} function}{0} or call the \doxysectlink{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2basic__json_2contains}{{\ttfamily contains} function}{0} before.

See also the documentation on \doxysectlink{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2element__access_2index}{element access}{0}.

??? example "{}\+Example 1:\+ Missing object key"{} \begin{DoxyVerb}The following code will trigger an assertion at runtime:

```cpp
#include <nlohmann/json.hpp>

using json = nlohmann::json;

int main()
{
    const json j = {{"key", "value"}};
    auto v = j["missing"];
}
```

Output:

```
Assertion failed: (m_value.object->find(key) != m_value.object->end()), function operator[], file json.hpp, line 2144.
```
\end{DoxyVerb}
\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2assertions_autotoc_md1393}{}\doxysubsection{\texorpdfstring{Constructing from an uninitialized iterator range}{Constructing from an uninitialized iterator range}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2assertions_autotoc_md1393}
Constructing a JSON value from an iterator range (see \doxysectlink{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2basic__json_2basic__json}{constructor}{0}) with an uninitialized iterator is undefined behavior and yields a runtime assertion.

using json = nlohmann::json;

int main()
{
    json::iterator it1, it2;
    json j(it1, it2);
}
```

Output:

```
Assertion failed: (m_object != nullptr), function operator++, file iter_impl.hpp, line 368.
```
\end{DoxyVerb}
\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2assertions_autotoc_md1394}{}\doxysubsection{\texorpdfstring{Operations on uninitialized iterators}{Operations on uninitialized iterators}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2assertions_autotoc_md1394}
Any operation on uninitialized iterators (i.\+e., iterators that are not associated with any JSON value) is undefined behavior and yields a runtime assertion.

using json = nlohmann::json;

int main()
{
  json::iterator it;
  ++it;
}
```

Output:

```
Assertion failed: (m_object != nullptr), function operator++, file iter_impl.hpp, line 368.
```
\end{DoxyVerb}

\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2assertions_autotoc_md1396}{}\doxysubsection{\texorpdfstring{Reading from a null {\ttfamily FILE} or {\ttfamily char} pointer}{Reading from a null {\ttfamily FILE} or {\ttfamily char} pointer}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2assertions_autotoc_md1396}
Reading from a null {\ttfamily \#!cpp FILE} or {\ttfamily \#!cpp char} pointer in C++ is undefined behavior. Until version 3.\+12.\+0, this library asserted that the pointer was not {\ttfamily nullptr} using a runtime assertion. If assertions were disabled, this would result in undefined behavior. Since version 3.\+12.\+0, this library checks for {\ttfamily nullptr} and throws a \href{../home/exceptions.md\#jsonexceptionparse_error101}{\texttt{{\ttfamily parse\+\_\+error.\+101}}} to prevent the undefined behavior.

??? example "{}\+Example 4:\+ Reading from null pointer"{} \begin{DoxyVerb}The following code will trigger an assertion at runtime:

```cpp
#include <iostream>
#include <nlohmann/json.hpp>

using json = nlohmann::json;

int main()
{
    std::FILE* f = std::fopen("nonexistent_file.json", "r");
    try {
        json j = json::parse(f);
    } catch (std::exception& e) {
        std::cerr << e.what() << std::endl;
    }
}
```

Output:

```
[json.exception.parse_error.101] parse error: attempting to parse an empty input; check that your input string or stream contains the expected JSON
```
\end{DoxyVerb}
