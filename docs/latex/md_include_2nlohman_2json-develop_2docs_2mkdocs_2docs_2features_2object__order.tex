\doxylink{requirements_The}{The} \href{https://tools.ietf.org/html/rfc8259.html}{\texttt{JSON standard}} defines objects as "{}an unordered collection of zero or more name/\+value pairs"{}. As such, an implementation does not need to preserve any specific order of object keys.\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2object__order_autotoc_md1493}{}\doxysection{\texorpdfstring{Default behavior:\+ sort keys}{Default behavior:\+ sort keys}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2object__order_autotoc_md1493}
\doxylink{requirements_The}{The} default type {\ttfamily \doxylink{namespacenlohmann_a8ba8c5483f73a36e716558571ff97d57}{nlohmann::\+json}} uses a {\ttfamily std::\+map} to store JSON objects, and thus stores object keys {\bfseries{sorted alphabetically}}.

??? example \begin{DoxyVerb}```cpp
#include <iostream>
#include "json.hpp"

using json = nlohmann::json;

int main()
{
    json j;
    j["one"] = 1;
    j["two"] = 2;
    j["three"] = 3;

    std::cout << j.dump(2) << '\n';
}
```

Output:

```json
{
  "one": 1,
  "three": 3,
  "two": 2
}
```
\end{DoxyVerb}
\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2object__order_autotoc_md1494}{}\doxysection{\texorpdfstring{Alternative behavior:\+ preserve insertion order}{Alternative behavior:\+ preserve insertion order}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2object__order_autotoc_md1494}

Alternatively, you can use a more sophisticated ordered map like \href{https://github.com/Tessil/ordered-map}{\texttt{{\ttfamily tsl::\+ordered\+\_\+map}}} (\href{https://github.com/nlohmann/json/issues/546\#issuecomment-304447518}{\texttt{integration}}) or \href{https://github.com/nlohmann/fifo_map}{\texttt{{\ttfamily nlohmann::\+fifo\+\_\+map}}} (\href{https://github.com/nlohmann/json/issues/485\#issuecomment-333652309}{\texttt{integration}}).\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2object__order_autotoc_md1495}{}\doxysubsection{\texorpdfstring{Notes on parsing}{Notes on parsing}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2object__order_autotoc_md1495}
Note that you also need to call the right \doxysectlink{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2basic__json_2parse}{{\ttfamily parse}}{0} function when reading from a file. Assume file {\ttfamily input.\+json} contains the JSON object above:\+


\begin{DoxyCode}{0}
\DoxyCodeLine{\{}
\DoxyCodeLine{\ \ "{}one"{}:\ 1,}
\DoxyCodeLine{\ \ "{}two"{}:\ 2,}
\DoxyCodeLine{\ \ "{}three"{}:\ 3}
\DoxyCodeLine{\}}

\end{DoxyCode}


!!! success "{}\+Right way"{} \begin{DoxyVerb}The following code correctly calls the `parse` function from `nlohmann::ordered_json`:

```cpp
std::ifstream i("input.json");
auto j = nlohmann::ordered_json::parse(i);
std::cout << j.dump(2) << std::endl;
```

The output will be:

```json
{
  "one": 1,
  "two": 2,
  "three": 3
}
```
\end{DoxyVerb}


??? failure "{}\+Wrong way"{} \begin{DoxyVerb}The following code incorrectly calls the `parse` function from `nlohmann::json` which does not preserve the
insertion order, but sorts object keys. Assigning the result to `nlohmann::ordered_json` compiles, but does not
restore the order from the input file.

```cpp
std::ifstream i("input.json");
nlohmann::ordered_json j = nlohmann::json::parse(i);
std::cout << j.dump(2) << std::endl;
```

The output will be:

```json
{
  "one": 1,
  "three": 3
  "two": 2,
}
```
\end{DoxyVerb}
