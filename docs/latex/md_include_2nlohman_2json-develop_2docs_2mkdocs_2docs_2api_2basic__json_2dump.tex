Serialization function for JSON values. \doxylink{requirements_The}{The} function tries to mimic Python\textquotesingle{}s \href{https://docs.python.org/2/library/json.html\#json.dump}{\texttt{{\ttfamily json.\+dumps()} function}}, and currently supports its {\ttfamily indent} and {\ttfamily ensure\+\_\+ascii} parameters.\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2basic__json_2dump_autotoc_md224}{}\doxysection{\texorpdfstring{Parameters}{Parameters}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2basic__json_2dump_autotoc_md224}
{\ttfamily indent} (in) :\+ If {\ttfamily indent} is nonnegative, then array elements and object members will be pretty-\/printed with that indent level. An indent level of {\ttfamily 0} will only insert newlines. {\ttfamily -\/1} (the default) selects the most compact representation.

{\ttfamily indent\+\_\+char} (in) :\+ \doxylink{requirements_The}{The} character to use for indentation if {\ttfamily indent} is greater than {\ttfamily 0}. \doxylink{requirements_The}{The} default is {\ttfamily  } (space).

{\ttfamily ensure\+\_\+ascii} (in) :\+ If {\ttfamily ensure\+\_\+ascii} is true, all non-\/\+ASCII characters in the output are escaped with {\ttfamily \textbackslash{}u\+XXXX} sequences, and the result consists of ASCII characters only.

{\ttfamily error\+\_\+handler} (in) :\+ how to react on decoding errors; there are three possible values (see \doxysectlink{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2basic__json_2error__handler__t}{{\ttfamily error\+\_\+handler\+\_\+t}}{0}:\+ {\ttfamily strict} (throws and exception in case a decoding error occurs; default), {\ttfamily replace} (replace invalid UTF-\/8 sequences with U+\+FFFD), and {\ttfamily ignore} (ignore invalid UTF-\/8 sequences during serialization; all bytes are copied to the output unchanged)).\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2basic__json_2dump_autotoc_md225}{}\doxysection{\texorpdfstring{Return value}{Return value}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2basic__json_2dump_autotoc_md225}
string containing the serialization of the JSON value\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2basic__json_2dump_autotoc_md226}{}\doxysection{\texorpdfstring{Exception safety}{Exception safety}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2basic__json_2dump_autotoc_md226}
Strong guarantee:\+ if an exception is thrown, there are no changes to any JSON value.\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2basic__json_2dump_autotoc_md227}{}\doxysection{\texorpdfstring{Exceptions}{Exceptions}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2basic__json_2dump_autotoc_md227}
Throws \href{../../home/exceptions.md\#jsonexceptiontype_error316}{\texttt{{\ttfamily type\+\_\+error.\+316}}} if a string stored inside the JSON value is not UTF-\/8 encoded and {\ttfamily error\+\_\+handler} is set to {\ttfamily strict}\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2basic__json_2dump_autotoc_md228}{}\doxysection{\texorpdfstring{Complexity}{Complexity}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2basic__json_2dump_autotoc_md228}

\begin{DoxyItemize}
\item "{}bytes"{}:\+ an array of bytes as integers
\item "{}subtype"{}:\+ the subtype as integer or {\ttfamily \#!json null} if the binary has no subtype
