\doxylink{requirements_The}{The} return value of each function determines whether parsing should proceed.

To implement your own SAX handler, proceed as follows:\+


\begin{DoxyEnumerate}
\item Implement the SAX interface in a class. You can use class {\ttfamily \doxylink{structnlohmann_1_1json__sax}{nlohmann::\+json\+\_\+sax}\texorpdfstring{$<$}{<}json\texorpdfstring{$>$}{>}} as base class, but you can also use any class where the functions described above are implemented and public.
\item Create an object of your SAX interface class, e.\+g. {\ttfamily my\+\_\+sax}.
\item Call {\ttfamily \#!cpp bool json::\+sax\+\_\+parse(input, \&my\+\_\+sax);} where the first parameter can be any input like a string or an input stream and the second parameter is a pointer to your SAX interface.
\end{DoxyEnumerate}

Note the {\ttfamily sax\+\_\+parse} function only returns a {\ttfamily \#!cpp bool} indicating the result of the last executed SAX event. It does not return {\ttfamily json} value -\/ it is up to you to decide what to do with the SAX events. Furthermore, no exceptions are thrown in case of a parse error -\/ it is up to you what to do with the exception object passed to your {\ttfamily parse\+\_\+error} implementation. Internally, the SAX interface is used for the DOM parser (class {\ttfamily json\+\_\+sax\+\_\+dom\+\_\+parser}) as well as the acceptor ({\ttfamily json\+\_\+sax\+\_\+acceptor}), see file {\ttfamily \doxylink{json__sax_8hpp_source}{json\+\_\+sax.\+hpp}}.\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2parsing_2sax__interface_autotoc_md1510}{}\doxysection{\texorpdfstring{See also}{See also}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2parsing_2sax__interface_autotoc_md1510}
