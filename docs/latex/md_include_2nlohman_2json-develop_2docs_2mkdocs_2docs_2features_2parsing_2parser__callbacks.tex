With a parser callback function, the result of parsing a JSON text can be influenced. When passed to {\ttfamily parse}, it is called on certain events (passed as {\ttfamily parse\+\_\+event\+\_\+t} via parameter {\ttfamily event}) with a set recursion depth {\ttfamily depth} and context JSON value {\ttfamily parsed}. \doxylink{requirements_The}{The} return value of the callback function is a boolean indicating whether the element that emitted the callback shall be kept or not.

\doxylink{requirements_The}{The} type of the callback function is:\+


\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keyword}{template}<\textcolor{keyword}{typename}\ BasicJsonType>}
\DoxyCodeLine{\textcolor{keyword}{using\ }parser\_callback\_t\ =}
\DoxyCodeLine{\ \ \ \ std::function<bool(\textcolor{keywordtype}{int}\ depth,\ parse\_event\_t\ event,\ BasicJsonType\&\ parsed)>;}

\end{DoxyCode}
\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2parsing_2parser__callbacks_autotoc_md1507}{}\doxysection{\texorpdfstring{Callback event types}{Callback event types}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2parsing_2parser__callbacks_autotoc_md1507}
We distinguish six scenarios (determined by the event type) in which the callback function can be called. \doxylink{requirements_The}{The} following table describes the values of the parameters {\ttfamily depth}, {\ttfamily event}, and {\ttfamily parsed}.


\begin{DoxyTable}{4}{}{}{1}
\SetCell{c,bg=\tableheadbgcolor,font=\bfseries}parameter {\ttfamily event}  &\SetCell{c,bg=\tableheadbgcolor,font=\bfseries}description  &\SetCell{c,bg=\tableheadbgcolor,font=\bfseries}parameter {\ttfamily depth}  &\SetCell{c,bg=\tableheadbgcolor,font=\bfseries}parameter {\ttfamily parsed}  \\
{\ttfamily parse\+\_\+event\+\_\+t::\+object\+\_\+start}  &the parser read {\ttfamily \{} and started to process a JSON object  &depth of the parent of the JSON object  &a JSON value with type discarded  \\
{\ttfamily parse\+\_\+event\+\_\+t::\+key}  &the parser read a key of a value in an object  &depth of the currently parsed JSON object  &a JSON string containing the key  \\
{\ttfamily parse\+\_\+event\+\_\+t::\+object\+\_\+end}  &the parser read {\ttfamily \}} and finished processing a JSON object  &depth of the parent of the JSON object  &the parsed JSON object  \\
{\ttfamily parse\+\_\+event\+\_\+t::\+array\+\_\+start}  &the parser read {\ttfamily \+[} and started to process a JSON array  &depth of the parent of the JSON array  &a JSON value with type discarded  \\
{\ttfamily parse\+\_\+event\+\_\+t::\+array\+\_\+end}  &the parser read {\ttfamily ]\+} and finished processing a JSON array  &depth of the parent of the JSON array  &the parsed JSON array  \\
{\ttfamily parse\+\_\+event\+\_\+t::\+value}  &the parser finished reading a JSON value  &depth of the value  &the parsed JSON value  \\
\end{DoxyTable}


??? example \begin{DoxyVerb}When parsing the following JSON text,

```json
{
    "name": "Berlin",
    "location": [
        52.519444,
        13.406667
    ]
}
```

these calls are made to the callback function:

| event          | depth | parsed |
| -------------- | ----- | ------ |
| `object_start` | 0     | *discarded* |
| `key`          | 1     | `#!json "name"` |
| `value`        | 1     | `#!json "Berlin"` |
| `key`          | 1     | `#!json "location"` |
| `array_start`  | 1     | *discarded* |
| `value`        | 2     | `#!json 52.519444` |
| `value`        | 2     | `#!json 13.406667` |
| `array_end`    | 1     | `#!json [52.519444,13.406667]` |
| `object_end`   | 0     | `#!json {"location":[52.519444,13.406667],"name":"Berlin"}` |
\end{DoxyVerb}
\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2parsing_2parser__callbacks_autotoc_md1508}{}\doxysection{\texorpdfstring{Return value}{Return value}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2features_2parsing_2parser__callbacks_autotoc_md1508}
Discarding a value (i.\+e., returning {\ttfamily \#!c false}) has different effects depending on the context in which the function was called:\+


\begin{DoxyItemize}
\item Discarded values in structured types are skipped. That is, the parser will behave as if the discarded value was never read.
\item In case a value outside a structured type is skipped, it is replaced with {\ttfamily \#!json null}. This case happens if the top-\/level element is skipped.
\end{DoxyItemize}
