This macro enables position diagnostics for generated JSON objects.

When enabled, two new member functions \doxysectlink{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2basic__json_2start__pos}{{\ttfamily start\+\_\+pos()}}{0} and \doxysectlink{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2basic__json_2end__pos}{{\ttfamily end\+\_\+pos()}}{0} are added to \doxysectlink{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2basic__json_2index}{{\ttfamily basic\+\_\+json}}{0} values. If the value was created by calling the\doxysectlink{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2basic__json_2parse}{{\ttfamily parse}}{0} function, then these functions allow querying the byte positions of the value in the input it was parsed from. In case the value was constructed by other means, {\ttfamily std::\+string::\+npos} is returned.

\doxysectlink{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2basic__json_2start__pos}{{\ttfamily start\+\_\+pos()}}{0} returns the position of the first character of a given value in the original JSON string, while \doxysectlink{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2basic__json_2end__pos}{{\ttfamily end\+\_\+pos()}}{0} returns the position of the character {\itshape following} the last character. For objects and arrays, the first and last characters correspond to the opening or closing braces/\+brackets, respectively. For primitive values, the first and last character represents the opening and closing quotes (strings) or the first and last character of the field\textquotesingle{}s numerical or predefined value ({\ttfamily true}, {\ttfamily false}, {\ttfamily null}), respectively.

Given the above, \doxysectlink{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2basic__json_2end__pos}{{\ttfamily end\+\_\+pos()}}{0}{\ttfamily  -\/ }\doxysectlink{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2basic__json_2start__pos}{{\ttfamily start\+\_\+pos()}}{0} for a JSON value provides the length of the parsed JSON string for that value, including the opening or closing braces, brackets, or quotes.

Note that enabling this macro increases the size of every JSON value by two {\ttfamily std::\+size\+\_\+t} fields and adds slight runtime overhead to parsing, copying JSON value objects, and the generation of error messages for exceptions. It also causes these values to be reported in those error messages.\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2macros_2json__diagnostic__positions_autotoc_md1187}{}\doxysection{\texorpdfstring{Default definition}{Default definition}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2api_2macros_2json__diagnostic__positions_autotoc_md1187}

!!! note "{}\+CMake option"{} \begin{DoxyVerb}Diagnostic positions can also be controlled with the CMake option
[`JSON_Diagnostic_Positions`](../../integration/cmake.md#json_diagnostic_positions) (`OFF` by default)
which defines `JSON_DIAGNOSTIC_POSITIONS` accordingly.
\end{DoxyVerb}


!!! note "{}\+Availability"{} \begin{DoxyVerb}Diagnostic positions are only available if the value was created by the [`parse`](../basic_json/parse.md) function.
The [`sax_parse`](../basic_json/sax_parse.md) function or all other means to create a JSON value **do not** set the
diagnostic positions and [`start_pos()`](../basic_json/start_pos.md) and [`end_pos()`](../basic_json/end_pos.md)
will only return `std::string::npos` for these values.
\end{DoxyVerb}


!!! warning "{}\+Invalidation"{} \begin{DoxyVerb}The returned positions are only valid as long as the JSON value is not changed. The positions are *not* updated
when the JSON value is changed.
\end{DoxyVerb}
