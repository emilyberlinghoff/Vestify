You can use the {\ttfamily nlohmann\+\_\+json::\+nlohmann\+\_\+json} interface target in CMake. This target populates the appropriate usage requirements for \href{https://cmake.org/cmake/help/latest/prop_tgt/INTERFACE_INCLUDE_DIRECTORIES.html}{\texttt{{\ttfamily INTERFACE\+\_\+\+INCLUDE\+\_\+\+DIRECTORIES}}} to point to the appropriate include directories and \href{https://cmake.org/cmake/help/latest/prop_tgt/INTERFACE_COMPILE_FEATURES.html}{\texttt{{\ttfamily INTERFACE\+\_\+\+COMPILE\+\_\+\+FEATURES}}} for the necessary C++11 flags.\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2integration_2cmake_autotoc_md1807}{}\doxysubsection{\texorpdfstring{External}{External}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2integration_2cmake_autotoc_md1807}
To use this library from a CMake project, you can locate it directly with \href{https://cmake.org/cmake/help/latest/command/find_package.html}{\texttt{{\ttfamily find\+\_\+package()}}} and use the namespaced imported target from the generated package configuration:\+

!!! example \begin{DoxyVerb}```cmake title="CMakeLists.txt"
cmake_minimum_required(VERSION 3.5)
project(ExampleProject LANGUAGES CXX)

find_package(nlohmann_json 3.12.0 REQUIRED)

add_executable(example example.cpp)
target_link_libraries(example PRIVATE nlohmann_json::nlohmann_json)
```
\end{DoxyVerb}


\doxylink{requirements_The}{The} package configuration file, {\ttfamily nlohmann\+\_\+json\+Config.\+cmake}, can be used either from an install tree or directly out of the build tree.\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2integration_2cmake_autotoc_md1808}{}\doxysubsection{\texorpdfstring{Embedded}{Embedded}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2integration_2cmake_autotoc_md1808}
To embed the library directly into an existing CMake project, place the entire source tree in a subdirectory and call {\ttfamily add\+\_\+subdirectory()} in your {\ttfamily CMake\+Lists.\+txt} file.

!!! example \begin{DoxyVerb}```cmake title="CMakeLists.txt"
cmake_minimum_required(VERSION 3.5)
project(ExampleProject LANGUAGES CXX)

# If you only include this third party in PRIVATE source files, you do not need to install it
# when your main project gets installed.
set(JSON_Install OFF CACHE INTERNAL "")

add_subdirectory(nlohmann_json)

add_executable(example example.cpp)
target_link_libraries(example PRIVATE nlohmann_json::nlohmann_json)
```
\end{DoxyVerb}


!!! note \begin{DoxyVerb}Do not use `#!cmake include(nlohmann_json/CMakeLists.txt)`, since that carries with it unintended consequences that
will break the build. It is generally discouraged (although not necessarily well documented as such) to use
`#!cmake include(...)` for pulling in other CMake projects anyways.
\end{DoxyVerb}
\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2integration_2cmake_autotoc_md1809}{}\doxysubsection{\texorpdfstring{Supporting Both}{Supporting Both}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2integration_2cmake_autotoc_md1809}
To allow your project to support either an externally supplied or an embedded JSON library, you can use a pattern akin to the following.

!!! example \begin{DoxyVerb}```cmake title="CMakeLists.txt"
project(ExampleProject LANGUAGES CXX)

option(EXAMPLE_USE_EXTERNAL_JSON "Use an external JSON library" OFF)

add_subdirectory(thirdparty)

add_executable(example example.cpp)

# Note that the namespaced target will always be available regardless of the import method
target_link_libraries(example PRIVATE nlohmann_json::nlohmann_json)
```

```cmake title="thirdparty/CMakeLists.txt"
if(EXAMPLE_USE_EXTERNAL_JSON)
    find_package(nlohmann_json 3.12.0 REQUIRED)
else()
    set(JSON_BuildTests OFF CACHE INTERNAL "")
    add_subdirectory(nlohmann_json)
endif()
```

`thirdparty/nlohmann_json` is then a complete copy of this source tree.
\end{DoxyVerb}
\hypertarget{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2integration_2cmake_autotoc_md1810}{}\doxysubsection{\texorpdfstring{Fetch\+Content}{Fetch\+Content}}\label{md_include_2nlohman_2json-develop_2docs_2mkdocs_2docs_2integration_2cmake_autotoc_md1810}
Since CMake v3.\+11, \href{https://cmake.org/cmake/help/v3.11/module/FetchContent.html}{\texttt{Fetch\+Content}} can be used to automatically download a release as a dependency at configure time.

!!! example \begin{DoxyVerb}```cmake title="CMakeLists.txt"
cmake_minimum_required(VERSION 3.11)
project(ExampleProject LANGUAGES CXX)

include(FetchContent)

FetchContent_Declare(json URL https://github.com/nlohmann/json/releases/download/v3.12.0/json.tar.xz)
FetchContent_MakeAvailable(json)

add_executable(example example.cpp)
target_link_libraries(example PRIVATE nlohmann_json::nlohmann_json)
```
\end{DoxyVerb}


!!! Note \begin{DoxyVerb}It is recommended to use the URL approach described above which is supported as of version 3.10.0. It is also
possible to pass the Git repository like

```cmake
FetchContent_Declare(json
    GIT_REPOSITORY https://github.com/nlohmann/json
    GIT_TAG v3.12.0
)
```

However, the repository <https://github.com/nlohmann/json> download size is quite large.
\end{DoxyVerb}
